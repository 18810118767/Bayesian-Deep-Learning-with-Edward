edward.inferences.metropolis_hastings module¶

class edward.inferences.metropolis_hastings.MetropolisHastings(latent_vars, proposal_vars, data=None, model_wrapper=None)[source]¶

Bases: edward.inferences.monte_carlo.MonteCarlo

Metropolis-Hastings.

Parameters:proposal_vars (dict of RandomVariable to RandomVariable) – Collection of random variables to perform inference on; each is binded to a proposal distribution p(z’ | z).

Examples

>>> z = Normal(mu=0.0, sigma=0.0)
>>> x = Normal(mu=z, sigma=1.0)
>>>
>>> qz = Empirical()
>>> proposal_z = Normal(mu=z, sigma)
>>> data = {x: np.array(5.0)}
>>> inference = ed.MetropolisHastings({z: qz}, {z: proposal_z}, data)

Notes

TODO Initialization is assumed from params[0, :]. This generalizes initializing randomly and initializing from user input. TODO assuming T x dim(var) and not dim(var) x T

No warm-up is implemented. Users must run MCMC for a long period of time, then manually burn in the Empirical random variable. (The complication here is how to allow warm-ups during the update() syntax.)

The updates assume Empirical().params is directly parameterized by tf.Variables(). It will also update with as many iterations as the minimum of all Empirical sizes.

build_update()[source]¶

Draw sample from proposal conditional on last sample. Then accept or reject the sample based on the ratio,

ratio = log p(x, znew) - log p(x, zold) +
log g(znew | zold) - log g(zold | znew)
= sum_z [ log p(znew) - log p(zold) +
log g(znew | zold) - log g(zold | znew) ] +

sum_x [ log p(x | znew) - sum_x log p(x | zold) ]
