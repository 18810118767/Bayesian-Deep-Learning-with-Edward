class edward.stats.distributions.Distribution¶

Bases: object

Template for all distributions.

entropy()¶
Parameters:params (float, np.array or tf.Tensor) – If univariate distribution, can be a scalar or vector. If multivariate distribution, can be a vector or matrix.
Returns:If univariate distribution, returns a tensor of same shape as input. If multivariate distribution, returns a tensor of shape[-1] from input: the outer dimension representing the multivariate dimension is collapsed.
Return type:tf.Tensor

Notes

The equivalent method in SciPy is not guaranteed to be supported for vector inputs for univariate distributions and matrix inputs for multivariate distributions. This is.

params as a 2d or higher tensor is not guaranteed to be supported for univariate distributions. params as a 3d or higher tensor is not guaranteed to be supported for multivariate distributions.

logpmf(x)¶
Parameters:
  • x (float, np.array or tf.Tensor) – If univariate distribution, can be a scalar or tensor. If multivariate distribution, can be a tensor; the outer dimension carries the multivariate dimension.
  • params (float, np.array or tf.Tensor) – scalar unless documented otherwise
Returns:

If univariate distribution, returns a tensor of same shape as input. If multivariate distribution, returns a tensor of shape[:-1] from input: the outer dimension representing the multivariate dimension is collapsed.
Return type:

tf.Tensor

Notes

x as a 3d or higher tensor is not guaranteed to be supported (for either univariate or multivariate distribution).

rvs(size=1)¶
Parameters:
  • size (int, list, or tuple, optional) – Number of samples, in a particular shape if specified in a list or tuple with more than one element.
  • params (float or np.array) –
Returns:

np.array of dimension (size x shape), where shape is the shape of its parameter argument. For multivariate distributions, shape may correspond to only one of the parameter arguments, e.g., alpha in Dirichlet, p in Multinomial, mean in Multivariate_Normal.
Return type:

np.ndarray

Notes

This is written in NumPy/SciPy, as TensorFlow does not support many distributions for random number generation. It follows SciPy’s naming and argument conventions. It does not support taking in tf.Tensors as input.

The equivalent method in SciPy is not guaranteed to be supported with a batch of parameter inputs, e.g., a vector of location parameters in a normal distribution, or a matrix of concentration parameters in a Dirichlet. This is.

This does not follow SciPy’s behavior, e.g., the number (or shape) of the draws will always be denoted by its outer dimension(s).

params as a 2d or higher tensor is not guaranteed to be supported (for either univariate or multivariate distribution).

size as a list or tuple of more than one element is not guaranteed to be supported.

For most distributions, the parameters must be of the same shape and type, e.g., n and p in Binomial must be np.arrays() of same shape or both floats. For some, they may differ by one dimension, e.g., n and p in Multinomial can be float and np.array(), or both np.arrays, and n always has one less dimension.
