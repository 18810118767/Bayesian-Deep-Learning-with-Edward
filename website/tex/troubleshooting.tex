\subsection{Troubleshooting}

\subsubsection{Basic Installation}

Edward depends on 
\begin{itemize}
  \item Numpy (>=1.7)
  \item Scipy (>=0.16)
  \item Progressbar (>=2.0)
  \item TensorFlow (>=0.7)
\end{itemize}

We recommend using \texttt{pip} to install \texttt{numpy}, \texttt{scipy}, and 
\texttt{progressbar} as
\begin{lstlisting}[language=JSON]
pip install numpy scipy progressbar
\end{lstlisting}

\href{https://www.tensorflow.org/}{TensorFlow} is not on PyPI yet. We recommend
using \texttt{pip} to install the latest TensorFlow binaries. Please
follow the 
\href
{https://www.tensorflow.org/versions/r0.9/get_started/os_setup.html#pip-installation}
{instructions} from Google.

\subsubsection{Full Installation}

Edward has optional features that depend on external packages.



\begin{itemize}
  \item The \href{http://mc-stan.org}{Stan} modeling language is supported
  through PyStan (>=2.0.1.3)
\begin{lstlisting}[language=JSON]
pip install pystan
\end{lstlisting}
  \item The \href{http://pymc-devs.github.io/pymc3/}{PyMC3} modeling language is supported
  through PyMC3 (>=3.0). Please see their installation
  \href{http://pymc-devs.github.io/pymc3/notebooks/getting_started.html}
  {instructions}. 
  \item Neural network modeling is supported through Keras (>=1.0)
\begin{lstlisting}[language=JSON]
pip install keras
\end{lstlisting}
  and PrettyTensor (>=0.5.3)
\begin{lstlisting}[language=JSON]
pip install prettytensor
\end{lstlisting}  
You have to manually \href{http://keras.io/#switching-from-theano-to-tensorflow}
{configure Keras} to work with TensorFlow.
\end{itemize}
